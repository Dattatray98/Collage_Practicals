# Practical 02: SQL Data Definition Language (DDL) Commands

## Objective
Write and execute SQL DDL commands such as `CREATE`, `ALTER`, `DROP`, `RENAME`, and `TRUNCATE` to define and modify tables. Apply integrity constraints such as `NOT NULL`, `UNIQUE`, `PRIMARY KEY`, `FOREIGN KEY`, and `CHECK`.

## Steps Performed
1.  **Creation**: Used `CREATE DATABASE` and `CREATE TABLE` to set up the Library database with constraints.
2.  **Constraints**: Applied `PRIMARY KEY`, `FOREIGN KEY` (with `ON DELETE CASCADE/SET NULL`), `NOT NULL`, and `CHECK` constraints.
3.  **Data Insertion**: Used `INSERT INTO` to populate initial records to verify constraints.
4.  **Schema Modification**: Demonstrated `ALTER` to add/modify columns (documented in the SQL script).

## Files in this Portfolio
- `DDL_Operations.sql`: Complete set of DDL and initial data commands.
