# Practical 01: Case Study and ER Diagram

## Objective
Develop a case study and design its Entity-Relationship (ER) Diagram. Convert the ER model into a relational model.

## Steps Performed
1.  **Requirement Analysis**: Identified the entities and their relationships for a Library Management System.
2.  **Case Study Documentation**: Wrote a detailed case study explaining the system's functionality.
3.  **ER Diagram Design**: Created an ER diagram using Mermaid notation to visualize entities (Student, Book, Author, etc.) and their relationships (Borrowing, Writing, Publishing).
4.  **Relational Model Conversion**: Converted the conceptual ER diagram into a logical relational schema (tables, primary keys, and foreign keys).

## Files in this Portfolio
- `CaseStudy.md`: Detailed functional requirements.
- `ER_Diagram.md`: Visual representation of the data structure.
- `Relational_Model.md`: Description of the resulting tables.
